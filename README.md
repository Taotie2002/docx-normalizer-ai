## 什么是DFGP？

DFGP = Document Format Generation Pattern
文档格式生成模式，包含：
- 字体规范（标题、正文、字号）
- 段落格式（行距、缩进、对齐）
- 页面设置（页边距、页眉页脚）
- 编号样式（多级列表、章节编号）

## 使用流程

### 核心概念

| 概念 | 说明 |
|------|------|
| **DFGP** | 文档格式生成模式，持久化积累，可迭代进化 |
| **目标文档结构** | 原文的段落层级、编号规律等，动态分析 |
| **灌注脚本** | 基于DFGP+目标结构生成的一次性脚本，仅对当前目标有效 |
| **焦土式策略** | 剥离原始格式，确保干净的石板上灌注新格式 |

### 多Agent分工

| Agent | 职责 | 思维特点 |
|-------|------|----------|
| **docx-analyzer** | 分析模板，抽取DFGP | 结构化、标准化思维 |
| **docx-scripter** | 分析目标结构，生成灌注脚本 | 实现导向、技术思维 |
| **docx-auditor** | 审计成品，发现问题 | 批判性思维、找问题 |
| **main** | 任务调度，不做执行 | 统筹协调 |

### 1. 模板入库（analyzer执行）
提供模板文档 → 识别类型 → 抽取DFGP → 存入知识库

### 2. 规格化任务（scripter执行）

```
目标文档
    ↓
[理解原文结构] ← 分析段落层级、编号规律、内容块
    ↓
[编写灌注脚本] ← 基于DFGP+目标结构，一次性脚本
    ↓
[焦土式剥离] ← 清除分栏、颜色、多字体、手动对齐等原始格式
    ↓
[执行脚本灌注] ← 在干净基础上应用DFGP格式
    ↓
规格化输出
```

### 3. 审计任务（auditor执行）

**审计评分阈值**：
- ≥ 90%：自动通过
- 80-90%：标记待确认
- < 80%：暂停，等待修正

**审计反馈循环**：
```
审计失败 (< 90%)
    ↓
[提交scripter] ← 问题清单
    ↓
[scripter修订脚本]
    ↓
[焦土式剥离] ← 再次清除残留格式
    ↓
[执行脚本灌注]
    ↓
[重新审计]
    ↓
循环直到 ≥ 90%
    ↓
提交用户
```

- 对比原文与输出，验证内容完整性
- 检查格式一致性（字体统一、缩进规范等）
- **原始文档必须保留**，审计时需要参考

### 4. 知识更新
遇到新特征 → 补充DFGP → 记录变更

## 审计维度（参考SRS设计）

### 精确匹配 (FATAL - 0容差)

- 发文字号、成文日期、发文机关标志

### 实体保留 (CRITICAL - 0容差)

- 机关名称、附件名称、主送机关列表

### 数值精度 (MAJOR)

- 总字数（容差5%）、金额/百分比、表格数量、图片数量

### 格式规范

- 纸张尺寸、页边距、字体、行数、字数
