## 什么是DFGP？

DFGP = Document Format Generation Pattern
文档格式生成模式，包含：
- 字体规范（标题、正文、字号）
- 段落格式（行距、缩进、对齐）
- 页面设置（页边距、页眉页脚）
- 编号样式（多级列表、章节编号）

---
> ⚠️ **最高原则**：GB/T 9704-2012《党政机关公文格式》是强制规范
> 
> 凡模板与该规范冲突的，**以该规范为准**，这是处理公文格式的**最高原则**！

## 使用流程

### 核心概念

| 概念 | 说明 |
|------|------|
| **DFGP** | 文档格式生成模式，持久化积累，可迭代进化 |
| **目标文档结构** | 原文的段落层级、编号规律等，动态分析 |
| **灌注脚本** | 基于DFGP+目标结构生成的一次性脚本，仅对当前目标有效 |
| **焦土式策略** | 剥离原始格式，确保干净的石板上灌注新格式 |

### 多Agent分工

| Agent | 职责 | Skill工具 |
|-------|------|----------|
| **钱串子 (finance)** | 分析模板，抽取DFGP | docx-analyzer |
| **代码驴 (coder)** | 生成灌注脚本 | docx-scripter |
| **审计狗 (auditor)** | 审计成品，发现问题 | docx-auditor |
| **main** | 任务调度，不做执行 | 统筹协调 |

### 1. 模板入库（钱串子执行）
提供模板文档 → 识别类型 → 抽取DFGP → 存入知识库

### 2. 规格化任务（代码驴执行）

```
目标文档
    ↓
[理解原文结构] ← 代码驴分析段落层级、编号规律、内容块
    ↓
[编写灌注脚本] ← 基于DFGP+目标结构，一次性脚本
    ↓
[焦土式剥离] ← main执行，清除分栏、颜色、多字体等原始格式
    ↓
[执行脚本灌注] ← 在干净基础上应用DFGP格式
    ↓
规格化输出
```

### 3. 审计任务（审计狗执行）

**审计评分阈值**：
- ≥ 90%：自动通过，提交用户
- 80-90%：标记待确认
- < 80%：**自动驱动返工循环**

**自动返工机制（main agent驱动）**：
```
审计失败 (< 90%)
    ↓
[main自动提交代码驴] ← 携带审计问题清单
    ↓
[代码驴修订脚本]
    ↓
[main执行焦土式剥离]
    ↓
[main执行脚本灌注]
    ↓
[main提交审计狗重新审计]
    ↓
循环直到 ≥ 90%
    ↓
[main自动提交用户]
```

**关键**：main agent必须自动执行返工循环，不需要等待用户驱动
```
审计失败 (< 90%)
    ↓
[提交代码驴] ← 问题清单
    ↓
[代码驴修订脚本]
    ↓
[焦土式剥离] ← 再次清除残留格式
    ↓
[执行脚本灌注]
    ↓
[重新审计]
    ↓
循环直到 ≥ 90%
    ↓
提交用户
```

- 对比原文与输出，验证内容完整性
- 检查格式一致性（字体统一、缩进规范等）
- **原始文档必须保留**，审计时需要参考

### 4. 知识更新
遇到新特征 → 补充DFGP → 记录变更

## 审计评分体系

### 综合评分权重（总分100）

| 审计维度 | 权重 | 评分标准 | GB/T 9704-2012要求 |
|---------|------|---------|-------------------|
| **字体使用** | **30%** | 正文=仿宋_GB2312，标题按级别对应黑体/楷体/方正小标宋 | 字体必须与段落属性严格匹配 |
| **对齐方式** | **20%** | 正文两端对齐，标题居中/左对齐 | 正文应两端对齐，标题居中 |
| **行间距** | **15%** | 固定值28pt或1.5倍行距，统一 | 段前段后间距统一 |
| **首行缩进** | **15%** | 正文32pt，标题0pt | 首行缩进2字符(32pt) |
| **页面设置** | **10%** | 3.7/3.5/2.8/2.6cm | 上3.7cm，下3.5cm，左2.8cm，右2.6cm |
| **内容完整性** | **10%** | 字符偏差<5% | 内容必须完整 |

### 字体属性识别规则（关键！）

| 段落属性 | 识别依据 | 正确字体 |
|---------|---------|---------|
| 主标题 | 文章大标题/文件名 | 方正小标宋简体 22pt |
| 一级标题 | 第X章 / X、格式 | 黑体 16pt |
| 二级标题 | X.Y / （一）格式 | 楷体_GB2312 16pt |
| 正文 | 非上述格式的普通段落 | 仿宋_GB2312 16pt |

> ⚠️ **重要**：属性识别错误会导致字体设置错误！脚本必须先准确识别段落属性，再应用对应字体。

### 行间距规范

| 类型 | GB/T 9704-2012要求 |
|------|-------------------|
| 正文章段落 | 固定值28pt 或 1.5倍行距 |
| 段前距 | 0pt |
| 段后距 | 0pt |
| 多倍行距 | 禁止（会导致不同设备显示不一致） |

### 精确匹配 (FATAL - 0容差)

- 发文字号、成文日期、发文机关标志

### 实体保留 (CRITICAL - 0容差)

- 机关名称、附件名称、主送机关列表

### 数值精度 (MAJOR)

- 总字数（容差5%）、金额/百分比、表格数量、图片数量
